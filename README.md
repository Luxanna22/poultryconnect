# 🐔 PoultryConnect 2.0

A web-based platform for smallholder poultry farmers to monitor farm operations, analyze production data, sell eggs directly to buyers, and connect with their farming community.

Built with **Flask** · **MySQL (XAMPP)** · **SQLAlchemy** · **Flask-Migrate**

## Setup Guide

### 1. Clone the Repository

```bash
git clone https://github.com/Luxanna22/poultryconnect.git
cd poultryconnect
```

---

### 2. Create a Virtual Environment

It is strongly recommended to use a virtual environment to isolate dependencies.

**Windows:**
```bash
python -m venv venv
venv\Scripts\activate
```

**macOS / Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
```

You should see `(venv)` in your terminal prompt when it is active.

---

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

**Dependencies installed:**

| Package | Purpose |
|---|---|
| `flask` | Core web framework |
| `flask-sqlalchemy` | ORM for database models |
| `flask-login` | User session management |
| `flask-migrate` | Database migration management (Alembic) |
| `flask-wtf` | Form handling and CSRF protection |
| `python-dotenv` | Loads `.env` variables automatically |
| `pymysql` | MySQL driver for Python |
| `email-validator` | Validates email format in forms |
| `pandas` | Data processing for analytics |
| `scikit-learn` | ML models for predictive analytics |
| `matplotlib` | Chart generation |
| `gunicorn` | Production WSGI server (Linux/Mac) |

---

### 4. Configure Environment Variables

Create a `.env` file in the root of the project by copying the example:

```bash
# Windows
copy .env.example .env

# macOS / Linux
cp .env.example .env
```

Then open `.env` and fill in your values:

```env
# Flask environment
FLASK_ENV=development

# Secret key — change this to a long random string in production
SECRET_KEY=your-secret-key-here

# MySQL Database URL
# Format: mysql+pymysql://<user>:<password>@<host>/<database_name>
# XAMPP default (no password):
DATABASE_URL=mysql+pymysql://root:@localhost/poultryconnect

# If you set a MySQL root password in XAMPP:
# DATABASE_URL=mysql+pymysql://root:yourpassword@localhost/poultryconnect
```

> ⚠️ **Never commit your `.env` file.** It is already listed in `.gitignore`.

---

### 5. XAMPP MySQL Setup

#### a) Start XAMPP Services

1. Open **XAMPP Control Panel**
2. Click **Start** next to **Apache**
3. Click **Start** next to **MySQL**

Both status lights should turn green.

#### b) Create the Database

1. Open your browser and go to: `http://localhost/phpmyadmin`
2. Click **New** in the left sidebar
3. Enter database name: `poultryconnect`
4. Set collation to: `utf8mb4_unicode_ci`
5. Click **Create**

#### c) (Optional) Set a MySQL Password

By default, XAMPP's MySQL root user has **no password**. If you want to set one:

1. In phpMyAdmin → click your username (root) in the top right
2. Go to **Change password**
3. Set your password
4. Update your `.env` file's `DATABASE_URL` accordingly

---

### 6. Run Database Migrations

These commands create all the tables in your `poultryconnect` database.

> Make sure XAMPP MySQL is **running** before executing these.

**First time setup — run all three:**

```bash
# Step 1: Initialize the migrations folder (only needed once)
flask db init

# Step 2: Generate migration scripts from your models
flask db migrate -m "initial tables"

# Step 3: Apply migrations to the database
flask db upgrade
```

**If you get `Target database is not up to date`:**

```bash
flask db upgrade        # sync first
flask db migrate -m "your message"
flask db upgrade        # apply new changes
```

**After pulling new code changes from the repo:**

```bash
flask db upgrade
```

#### Tables Created

| Table | Description |
|---|---|
| `alembic_version` | Tracks migration history (managed by Flask-Migrate) |
| `users` | All user accounts across all roles |
| `farms` | Poultry farms (a farmer can register multiple) |
| `production_records` | Daily egg production and feed logs per farm |
| `expenses` | Operational expense entries per farm |

---

### 7. Run the Application

```bash
python run.py
```

The app will start on: **http://127.0.0.1:5000**

Open your browser and visit the URL. The landing page will load automatically.

> To run in debug mode (auto-reload on file changes), ensure `FLASK_ENV=development` is set in your `.env`.

## User Roles

| Role | Description |
|---|---|
| `admin` | Manages users, farms, and system reports |
| `farmer` | Manages farms, logs production, sells on marketplace |
| `buyer` | Browses and orders eggs from farms |
| `feed_supplier` | Listed in community directory, receives messages |
| `veterinarian` | Listed in community directory, receives messages |

Role is selected during registration and determines which dashboard and features are accessible.

## Common Issues

### `ModuleNotFoundError: No module named 'pymysql'`
You have not installed dependencies or your virtual environment is not active.
```bash
venv\Scripts\activate   # Windows
pip install -r requirements.txt
```

### `Access denied for user 'root'@'localhost'`
Your MySQL password in `.env` does not match XAMPP. Either remove the password from XAMPP or update your `DATABASE_URL` in `.env`.

### `Target database is not up to date`
Run `flask db upgrade` before running `flask db migrate`. See [Step 6](#6-run-database-migrations).

### `flask: command not found`
Your virtual environment is not activated, or Flask is not installed.
```bash
venv\Scripts\activate
pip install flask
```

### Port 5000 already in use
Another process is using port 5000. Run on a different port:
```bash
flask run --port 5001
```
